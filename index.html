π0.7: a steerable model with emergent capabilities

Physical Intelligence trained one robot policy that can fold laundry on a robot body it was not trained on, take language coaching for an unfamiliar air fryer, and perform dexterous skills that previously needed per-task specialists.

This page starts with the basics, then builds to the key π0.7 idea: train with diverse context so the same model can understand not just what to do, but how to do it.

Part 1 · foundations before robotics jargon
01 · Policy

A policy turns what the robot sees into what it does.

A robot policy is a function. Its input is an observation: camera pixels, joint angles, and the current instruction. Its output is an action: how the robot should move next.

In the toy arm, you choose the next joint angles by hand. In π0.7, a neural network chooses continuous chunks of joint motion many times per second.

The whole π-family can be read as learning $\pi(a \mid o, c)$: actions from observations plus context.

02 · Demonstrations

Imitation learning works where the demos cover the world.

Behavior cloning records demonstrations and trains a model to predict the demonstrated action from the observation. It is powerful, but it depends on coverage.

Click to add demos. The fitted curve is confident near examples and shaky away from them. Real robot learning has the same problem, except the space is camera pixels, objects, bodies, and long-horizon tasks.

The challenge is not memorizing common motions. It is staying useful when the scene, task, or robot body shifts.

03 · Multimodality

One image can have many correct actions.

Human demonstrators do not move identically. One reaches high, another curls around an obstacle, and both can be right. If a model averages those choices, the result can be a motion no demonstrator actually used.

This is why π-family models do not just regress one action. They produce continuous action chunks using flow matching, which lets multiple action modes remain separate.

Robot action data is multimodal. Good policies need to represent several plausible futures, not an average future.

Three valid trajectories

Three paths reach the same target. Averaging them would trace a path that belongs to none of the demonstrations.

04 · Continuous action chunks

Flow matching turns noise into a sampled action trajectory.

Instead of picking one averaged action, a flow-matching policy learns a vector field over action space. At inference time, it starts from noise and follows the field toward a plausible action chunk.

The toy visualization is two-dimensional. π0.7 uses the same idea in a much larger action space, conditioned on images, language, and context.

Flow matching is the generative piece that makes continuous, multimodal robot motion practical.

05 · Vision-language-action

π0.7 is a VLA: vision and language in, robot actions out.

A vision-language model already knows visual and semantic concepts: drawers, handles, shirts, buttons. A vision-language-action model adds the ability to output robot motion.

π0.7 keeps the image-language backbone and attaches an action expert. The interesting part is how PI conditions that expert so one checkpoint can work across many data sources.

The language model analogy matters: semantic knowledge is useful only if it can be grounded into physical actions.

Part 2 · why broad robot data needs context
06 · Naive mixing

Dumping every robot's data together creates contradictions.

The obvious recipe is to mix demonstrations from every robot and train one model. PI says this does not work well by itself.

Different robots can see similar scenes but need different actions because their bodies, grippers, control interfaces, and strategies differ. Without context, the model sees conflicting labels.

A generalist needs the data, but it also needs the labels that explain how each behavior should be interpreted.

Off: the model averages conflicting demonstrations. On: context separates the modes.

07 · Diverse contextual prompting

The central move: tell the model what to do and how to do it.

System map · adapted from PI's public blog figure

This redraws the blog diagram as a responsive guide. Decorative video thumbnails use PI's public demo clips; the source blog and paper remain the source of truth.

PI's phrase is diverse context. During training, examples can be conditioned on language, metadata, control modality, and visual subgoals. These context pieces disambiguate broad data instead of forcing the model to average it away.

At test time, the same interfaces become controls. You can ask for a subtask, request a strategy, or provide a generated visual goal.

Context makes diverse robot data trainable and makes the resulting model steerable.

08 · Visual subgoals

A world model sketches the next scene; π0.7 acts toward it.

Visual subgoals are the most concrete context channel. A lightweight world model predicts what the scene should look like next, and the action policy is conditioned on that target image.

This separates high-level intent from low-level control: first imagine the desired near-future pixels, then choose the joint motion that moves the real scene there.

For manipulation, a picture of the desired next state can be a clearer instruction than words alone.

Part 3 · evidence from the release
09 · Compositional generalization

The air fryer shows why steerability matters.

The air fryer is not a task PI says was directly demonstrated in the evaluation setup. The model has related skills: open hinged things, grasp handles, place objects, press buttons.

Plain language gets partway there. More context, such as coarse steps, detailed steps, or a high-level policy plus visual subgoals, makes the same π0.7 policy far easier to steer.

The compelling behavior is not a magic new primitive. It is recombination: known skills arranged for an unfamiliar appliance.

10 · Cross-embodiment transfer

The same checkpoint folds laundry on a new robot body.

PI's UR5e example tests more than language following. The robot body changes: size, inertia, gripper geometry, and teleoperation difficulty all shift.

The paper reports no laundry-folding data for this task on the bimanual UR5e system. π0.7 still transfers the behavior and adapts the physical strategy.

This is the robotics version of an analogy: map a familiar task onto a body that solves it differently.

12 · Limits

π0.7 is a step-change, not a finish line.

PI is careful about the boundary: unseen tasks remain harder than seen tasks, and broad datasets make it difficult to prove that a behavior is entirely novel rather than a new combination of related experience.

The paper also points toward a future model that can plan, act, observe, and revise. π0.7 is steerable, but explicit reflective planning is still a next problem.

The next frontier is turning semantic reasoning into reliable physical reasoning over long horizons.

Open problems called out by the work
Novel tasksZero-shot and compositional tasks are still less reliable than familiar task families.
Novelty auditWith large robot datasets, deciding whether a behavior is truly unseen is technically difficult.
Reflective planningFuture systems need to reason through plans, see outcomes, and revise before acting again.
