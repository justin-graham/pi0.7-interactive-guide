Foundational ideas
01 · Policy

A policy is the rule for choosing the next move.

Write a policy as $\pi(a \mid o, c)$: choose an action $a$ from an observation $o$ and optional context $c$. The observation might be camera pixels and joint angles. The context might be a language instruction, robot type, or visual goal.

In this interactive arm, you are the policy. You see the target and choose two joint angles. A learned robot policy does the same kind of mapping, but in a much larger action space.

03 · Behavior cloning

Imitation learning copies a demo's actions.

Behavior cloning records demonstrations and trains a model to predict the demonstrated action from the observation. It is the simplest bridge from data to policy.

Click to add demos. The fitted curve is confident near examples and shaky away from them. In robotics, "near" means similar camera views, object poses, robot states, and task context.

Imitation is powerful where the demonstrations cover the situation. It becomes brittle when the robot drifts into states no one demonstrated.

04 · Covariate shift

Small imitation errors matter.

A policy trained only on expert states may not know what to do after its own mistake. One slight miss changes the next observation, which can cause the next action to be worse.

The yellow band is the demonstration corridor. The learned policy is fine inside it, but becomes less reliable as the state drifts away.

This is why broad data and good context matter: the robot has to act in the real world.

05 · Multimodality

One observation has several correct futures.

Human demonstrators do not move identically. One reaches high, another curls around an obstacle, and both can be right. If a model averages those choices, the result can be a motion no demonstrator actually used.

08 · Naive mixing

Physical Intelligence's recipe is to mix demonstrations from every robot and train one policy. The problem is that similar observations can require different actions because robot bodies, grippers, control interfaces, and strategies differ.

Turn on the context label. The same data stops looking contradictory because the model is told which behavior mode it should imitate.

A generalist needs broad data, but it also needs context that explains how each behavior should be interpreted.

09 · Context coordinates

Think of context as coordinates. Language says the semantic goal. Embodiment or control metadata says which robot and which interface. Quality and speed tags say which style of behavior to prefer. Visual subgoals say where the scene should go next. PI's phrase for π0.7 is diverse context.

A steerable model is a conditional model: change the coordinates, change the behavior without changing the checkpoint.

10 · Visual subgoals

Some spatial instructions are easier to show than to say. A lightweight world model can predict what the scene should look like next, and the action policy can act toward that target image.

This separates high-level intent from low-level control: first imagine the desired near-future pixels, then choose the joint motion that moves the real scene there.

Visual subgoals are context too. They steer the policy with pixels instead of only words.

15 · Limits

Unseen tasks remain harder than familiar task families, and broad datasets make it difficult to prove that a behavior is entirely novel rather than a new combination of related experience. The next frontier is a policy that can plan, act, observe, and revise over long horizons with the same reliability that it executes short chunks.

Open problems
Novel tasksZero-shot and compositional tasks are still less reliable than familiar task families.
Novelty auditWith large robot datasets, deciding whether a behavior is truly unseen is technically difficult.
Reflective planningFuture systems need to reason through plans, see outcomes, and revise before acting again.
